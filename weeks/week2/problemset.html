🖥️ System Requirements & Compute Strategy

To successfully navigate the Keras 3 and JAX paradigm, you must understand *where* and *how* your code will run. We will use a hybrid approach:

1. Local Environment (Week 2: Prototyping & CI/CD)

  • Use Case: Setting up high-speed repositories, interacting with AI agents, writing Python scripts, and generating mathematical datasets.
  • Hardware: Standard laptop (Windows/macOS/Linux), min. 8GB RAM. A standard CPU is perfectly fine for these tasks.
  • The Modern Stack (uv): Legacy engineers wait minutes for pip to resolve dependencies. We will use uv (by Astral), the ultra-fast Python package installer and resolver. It natively handles virtual environments and deterministic lockfiles.
  • Setup: Code Editor (VS Code or Cursor), Git installed locally, and uv installed globally:
macOS / Linux:
curl -LsSf https://astral.sh/uv/install.sh | sh
Windows:
powershell -ExecutionPolicy ByPass -c "irm https://astral.sh/uv/install.ps1 | iex"

Note: You can also ask your coding agent to install it for you!

2. Google Colab (Week 3: Massive Compute & XLA Compilation)

  • Use Case: Training the Deep Autoencoder and compiling JAX arrays.
  • Hardware: Cloud-based NVIDIA T4 GPUs or Google TPUs (v2-8).
  • Why Colab? Installing GPU/TPU drivers locally is notoriously frustrating. Colab provides free, instant access to hardware accelerators. You will pull your GitHub repository into Colab and use uv pip in the cloud for blistering-fast dependency installation.

⚡ Exercise 1: The Vault of Version Control & Blazing Init (uv & Git)

*Learning Objective: Initializing projects with* *uv, managing* *.gitignore, and deploying to GitHub using Agentic workflows.*

  1. Phase 1 (Create the Remote Vault): Go to github.com and create a new repository named genesis-oracle (or give your own name\!) Set visibility to Public (visibility: public). Do *not* initialize it with a README or .gitignore.

Phase 2 (The Local Ignition): Open your terminal. Instead of manually creating folders and virtual environments, let uv scaffold your project instantly:

uv init genesis-oracle
cd genesis-oracle

*Notice how* *uv init* *automatically creates a* *pyproject.toml, a* *.venv, and an initial* *hello.py* *script\!*

  1. Phase 3 (The Mighty Bouncer): Open the .gitignore file that uv created. It already ignores .venv/. Manually add your data directory to the bottom: data/ and __pycache__/ (we never upload heavy, raw datasets to GitHub). Create a data/ and src/ folder.
  2. Phase 4 (The Agentic Push): A modern engineer delegates boilerplate commands. Open your Gemini CLI (or Antigravity ADE) and prompt your agent (as always you can use your own prompts\!):
    3. My remote URL is [YOUR_GITHUB_URL]. Please write the terminal commands to initialize git, stage all files respecting the .gitignore, create a commit with the message 'Initial Genesis Vault setup', and push it to the main branch.
  3. Phase 5 (Go Live): Convert your Agent_Report.md from Week 1 into an index.md file and push it. Go to your GitHub Repository Settings \> Pages. Under "Build and deployment", select the main branch as the source and click Save. Submit your live GitHub Pages link. ⚡ Exercise 2: Aligning the Triad (Keras 3 + JAX via uv add)

*Learning Objective:Modern dependency resolution and Keras 3 backend configuration.*

  1. The Installation: Standard pip is obsolete here. Use uv to instantly lock and install your ML stack into your pyproject.toml:
uv add "keras>=3.0.7" jax jaxlib numpy scipy matplotlib

*Watch how* *uv* *resolves and installs gigabytes of ML libraries in a fraction of the time, automatically locking exact dependencies in your* *uv.lock* *file.*

  1. Environment Configuration: Create a new Python script src/oracle_setup.py. Before importing Keras, use Python's os module to set the environment variable "KERAS_BACKEND" to "jax".
  2. Verification: Verify the backend is active by writing code to create a basic Keras random tensor and printing its type. Run it safely inside the environment using:
uv run python src/oracle_setup.py
  1. Deliverables: Use Git to commit and push pyproject.toml, uv.lock, and oracle_setup.py to GitHub.

⚡ Exercise 3: Synthesis of the Aether (Fourier Series & RC Filters)

*Learning Objective: Data-driven modeling and synthetic time-series generation.*

  1. The Fourier Datastream: Let \(T\) be the last two digits of your student ID, and \(\omega_0 = 2\pi/T\) . Write a Python script (src/data_generator.py) to generate the continuous Fourier series of a square wave over 100 periods using the first 9 odd harmonics.
  2. The Numerical RC Filter: Calculate the analytical effect of an RC low-pass filter on your signal:

\(R = 0.5 \text{ k}\Omega\) ,

\(C = 1000 + \text{last 3 ID digits } \mu\text{F}\) .

Calculate the amplitude and phase shift for each harmonic using

\(H(\omega) = \frac{1}{1 + j\omega RC}\) .

Generate your filtered baseline signal.

  1. Noise & Sabotage (Anomaly Injection): Add random Gaussian noise to your signal. Then, artificially corrupt the signal: between period 70 and 75, inject a massive high-frequency voltage spike. Save this 1D continuous array locally inside your /data folder.
  2. Deliverables: Run your code (uv run python src/data_generator.py). Plot a window of your normal noisy signal next to the anomaly spike. Save the plot as data_feed.png and push your generator code and the plot to GitHub.
