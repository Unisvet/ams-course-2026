⚡ Exercise 1: Forging the Digital Sanctum (Environment & Terminal Automation)

*Learning Objective: Setup of the Python/AI environment and automation of terminal workflows using the Gemini CLI.*

Before an Architect can command agents, they must prepare their workspace. Instead of clicking through operating system menus, you will use AI to orchestrate your system via natural language.

  1. The Python Uplink: Create an isolated Python 3.10+ virtual environment named nexus_env and activate it. Install the core simulation artifacts: numpy, scipy, and matplotlib.
  2. Summoning the Oracle: Install the [gemini-cli](https://geminicli.com/) and authenticate it with your API key. Test your connection by asking it directly from the terminal:
    3. gemini-cli "In one sentence, what is the paradigm shift from classical block-diagram simulation to Agentic Engineering?"
  3. Agentic Bash Automation: A true Architect automates the mundane. Do not create your project directories by hand\! Prompt the Gemini CLI to generate a bash script (init_nexus.sh) that builds your workspace.

  • Prompt Example: 
    Write a bash script that creates a modern Python project structure with directories src/, data/, agents/, and docs/. Inside src/, create an empty main.py. Output only the executable bash code.

  1. Deliverables: Execute the script. Provide a screenshot of your terminal showing your Gemini prompt, the execution of the bash script, and the resulting directory tree (using the tree command).

⚡ Exercise 2: Echoes of Electronica (Continuous Systems & Dimensionless Variables)

*Learning Objective: Translating continuous systems into Python; understanding dimensionless variables.*

Deep in the archives, you uncover the ancient differential equations once solved by the Sorcerers of Electronica:

  • Harmoniously swinging pendulum: \(\ddot{x} + \omega^2x = 0\) , with \(x(0)=0, \dot{x}(0)=1, \omega>0\)
  • Radioactive decay: \(\dot{x} = -\alpha x\) , with \(x(0)=1, \alpha>0\)

Today, we define physics in pure code.

  1. Dimensionless Awakening (Theory): In the lecture, we discussed the power of dimensionless variables. Transform the pendulum equation into a dimensionless form on paper using a new time variable \(\tau = \omega t\) .

  • Question: Why is transforming physical states into dimensionless variables a critical superpower when training Neural Networks and scaling up models on JAX/Flax later in the course?

  1. Pair-Programming with the AI: Instead of searching for MATLAB's dsolve, you will pair-program. Prompt the Gemini CLI to write a highly documented Python script (src/ancients.py). Instruct the AI to solve *both* continuous differential equations using scipy.integrate.solve_ivp (use \(\omega=2\) , \(\alpha=0.5\) ) and plot the results side-by-side using matplotlib for \(t \in [0, 10]\) .
  2. Deliverables: Submit your manual dimensionless derivation, the answer to the ML question, the exact prompt you used, and the final plotted image.

⚡ Exercise 3: The Pulse of Time (Discrete vs. Continuous)

*Learning Objective: Diskrete vs. kontinuierliche Systeme; understanding Simulation Chains.*

The machines of the Neo-Simulacrum operate in discrete computational steps, but the physical universe flows continuously. Let us test the boundaries of simulation using a magical conduit from the "Journey Through the Circuit Realm."

  • The RL Circuit: Governed by \(I'(t) = V(t) - 0.2 I(t)\) , where the voltage is \(V(t) = 5\sin(t)\) and \(I(0)=0\) .

  1. Two Paths: Ask your Gemini assistant to modify your Python script to simulate this RL circuit continuously (using solve_ivp) AND discretely (using an explicit Euler step method \(I_{k+1} = I_k + (V_k - 0.2 I_k) \Delta t\) ).
  2. Break the Matrix: Plot both lines on the same graph. Run the code with a precise step size of \(\Delta t = 0.1\) . Now, deliberately sabotage the simulation by changing the discrete time step to a very large value (e.g., \(\Delta t = 11\) ).
  3. Deliverables: Capture the distorted plot. Write a short paragraph explaining *why* the discrete model fails catastrophically at large step sizes while the continuous solver remains stable. Relate this briefly to the concept of "Simulationsketten" (Simulation chains) from the lecture.

If it exists, write a markdown file named docs/Agent_Report.md summarizing what physical systems the script simulated and confirming the successful execution."*
