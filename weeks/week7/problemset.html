⚡ Exercise 3: Parameter Hide-and-Seek (Structured JSON Modification)

*Learning Objective: Enforcing type-safe schemas to drive automated parameter search loops.*

Create a simulated physical environment script src/sandbox_env.py that models a simple thermal dampener. The dampener relies on an internal system parameter \(\kappa\) (Kappa). If \(\kappa\) is too low, the system freezes; if \(\kappa\) is too high, it explodes.

Define a strict structured contract using **Pydantic** to capture the model's regulatory decisions:

from pydantic import BaseModel, Field

class ControlDecision(BaseModel):
    system_state: str = Field(description="Must be 'FREEZING', 'BOILING', or 'PERFECT'")
    adjustment_action: str = Field(description="Must be 'INCREASE', 'DECREASE', or 'HOLD'")
    delta_value: float = Field(description="The exact numerical change to apply to Kappa")
    confidence_score: float

The Game Loop: Write a script src/game_loop.py that forces Gemini into a closed loop with your sandbox environment. For 5 consecutive turns, feed the model the current temperature log via the API, enforce the ControlDecision response schema, parse the JSON payload programmatically, and actively modify \(\kappa\).

The objective is to guide a highly volatile starting state into the "PERFECT" zone within 5 iterations.

Deliverables: Submit the complete script execution logs showing the structured JSON tokens exchanged at every single turn of the loop.
