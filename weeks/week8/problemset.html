⚡ Exercise 3: The Discrete Queue Simulation (Foundry)

*Learning Objective: Stochastic simulation of discrete events.*

Parts arrive, wait in a buffer, and are processed by a machine. We simulate this behavior stochastically. Arrival times follow a Poisson process with rate \(\lambda\) (Arrival Rate), while processing times at the station follow a stochastic distribution with rate \(\mu\) (Service Rate).

  1. Class DiscreteFoundry: Write a simulation class representing the state of a buffer.
  2. Instability: If the arrival rate exceeds the service rate, the buffer fills up.
  3. Overflow: Upon reaching maximum capacity, a buffer overflow occurs (data loss/scrap).
  4. Step Function: The step() method must update system variables and export a JSON string containing the metrics.
